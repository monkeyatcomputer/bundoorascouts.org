# Data & Collections

This document covers how data files and collections are used in the Bundoora Scouts repository.

## Data Files (`_data/`)

Data files allow for easy updates to content without changing HTML structure.
*   `navigation.yml`: Defines the main navigation menu.
*   `footer.yml`: Defines footer links.
*   `sections.yml`: Data for scouting sections (Joeys, Cubs, etc.).
*   `sponsors.yml`: Data for sponsors.
*   **Trip Data**: Folders inside `_data/` (e.g., `_data/ki-2023/`) contain data for specific trips. Files like `route.yml` and `telemetry.yml` in these folders are generated by scripts (see Scripts section in Architecture) and should not be edited manually.

## Collections

Collections are groups of related documents.
*   **Posts**: Standard Jekyll blog posts located in `_posts/`.
*   **People**: Located in `_people/`. Used for listing leaders or members.
*   **Trips**: Located in `_trips/`. Used for detailed expedition pages (e.g., Kangaroo Island 2023). They use complex frontmatter to link to data in `_data/`.
